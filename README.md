# faham

**在时间里看见政策** —— 一个非营利、非政党的马来西亚公民事实档案。

以可缩放的时间轴记录公共政策：什么政策、谁负责、进展如何、证据在哪。
不替任何一方说话，只把事件、来源与各方立场并列摆出来，让读者自己判断。

---

## 为什么开源

这个平台的全部价值建立在「可信」两个字上。而闭源的事实平台要求读者无条件信任运营者——
这本身就与它的主张矛盾。开源意味着任何人都可以验证：

- 没有埋点、没有追踪脚本、没有第三方请求
- 排序与筛选逻辑是公开的，无法暗中偏袒某一方
- 每一次内容修改都留在 git 历史里，改了什么、什么时候改的，都查得到

**「留痕不删」不只是页面上的一句承诺，它就是这个仓库的提交历史。**

---

## 内容准则

1. **每条主张都要有来源。** 没有可靠来源的数字，不发布——宁可留空并注明「待补充来源」。
2. **正反并列。** 争议性事件必须同时呈现支持与批评方的说法，各自附来源。
3. **状态变更留痕。** 政策口径变了就在 `hist` 里追加一条，不静默改写历史。
4. **用数据说话，不按身份分栏。** 差异化影响以有来源的数据呈现，不做族群导航分类。
5. **发现错误请提 issue 或 PR。** 纠错记录本身也是公开的。

## 技术

单文件静态页面，零依赖、零构建、零后端：

- 直接双击 `index.html` 即可本地查看
- 部署 = 把文件夹上传到任何静态托管（Cloudflare Pages / Netlify / GitHub Pages）
- 明亮（档案馆）／暗色（观测台）双主题，跟随系统或手动切换
- 时间轴支持二维自由拖拽，缩放粒度自动适配：年 → 季 → 月 → 日

## 新增或修改事件

编辑 `index.html` 顶部的 `EVENTS` 数组即可，字段说明：

| 字段 | 说明 |
|---|---|
| `date` | `YYYY-MM-DD`，用于排序与定位 |
| `label` | 显示用日期文字 |
| `title` | 事件标题 |
| `topic` | 议题分类：`subsidy` / `tax` / `cost` / `vote` / `graft` |
| `w` | 权重 1–3，决定光点大小 |
| `who` / `why` | 责任方 / 起因 |
| `hist` | 状态历程 `[{d:"时间", t:"变化"}]` |
| `claims` | 各方主张 `[{side, who, tx, sr, url}]`，`side` 为 `for`/`against`/`neutral` |
| `impact` | 差异化影响 `[{k, v, s}]` —— **只放有来源的数据** |
| `note` | 附注／提醒 |
| `srcs` | 主要来源 `[{l:"标题", u:"链接"}]` |
| `img` | 图片 `[{cap:"图注", src:"相对路径"}]` |
| `discuss` | 讨论条数（阶段2接后端后才真实） |

**图片请自托管**（放进本仓库、用相对路径），不要外链——外链会让读者的浏览器去请求第三方服务器。
优先使用自制图表：无版权风险、无第三方请求。

## 路线图

- **阶段 1（当前）** 只读时间线，验证「结构本身有价值」
- **阶段 2** 信任分层评论系统（需后端与审核流程；这是法律风险最集中的一环，不急）
- **阶段 3** 政党自呈政绩 + 社群核查者体系
